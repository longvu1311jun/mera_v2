Toàn bộ các tính năng và API endpoints trong dự án MeraGroup v2

📌 Menu Chính — Trang Giao Diện
🏠 Dashboard / Trang Chủ PAGE
GET /

Trang chính. Nếu chưa đăng nhập → login Lark OAuth. Nếu đã đăng nhập → dashboard tổng quan.

📊 Báo Cáo Sale 1 PAGE
GET /saleReport

Thống kê tin nhắn, đơn chốt, tỉ lệ hủy của từng tư vấn viên. Hỗ trợ export Excel.

📈 Thống Kê Lịch Hẹn PAGE
GET /stats

KPI lịch hẹn CSKH: tổng khách, tổng lịch, hoàn thành, muộn/quá hạn.

📝 Logs Realtime PAGE
GET /logs

Xem log hệ thống realtime — webhook events, sync jobs, token refresh.

📋 Report (Legacy) PAGE
GET /report

Báo cáo cũ lấy từ Lark Bitable — tóm tắt Sales Summary theo bảng.

🔎 Tra Cứu & POS
🔎 Tìm Khách Hàng 360° PAGE
GET /search-info

Tra cứu khách hàng theo SĐT từ Pancake POS + Lark Bitable. Hiển thị đơn hàng, ghi chú, lịch hẹn.

🕵️ Tra Cứu Của SALE PAGE
GET /searchInfo

Giao diện tra cứu dành riêng cho Sale — tìm thông tin khách từ Bitable.

🔍 Tra Cứu Khách Hàng (v1) PAGE
GET /tra_cuu_khach_hang

Tìm kiếm khách hàng song song trên nhiều Base Lark theo SĐT.

📦 Danh Sách Đơn POS PAGE
GET /pancake-orders

Xem danh sách đơn hàng từ Pancake POS đã đồng bộ vào DB nội bộ.

🔄 Đồng Bộ Dữ Liệu PAGE
GET /dongbo

Trang quản lý đồng bộ: đơn hàng, sản phẩm, khách hàng, nhân viên từ POS/Lark.

🧪 Công Cụ Sync POS PAGE
GET /test-api.html

UI để force sync đơn hàng từ Pancake POS vào DB theo khoảng thời gian tùy chọn.

⚙️ Hệ Thống & Cấu Hình
⚙️ Cấu Hình Webhook PAGE
GET /config

Quản lý webhook status 1 & 6, xem danh sách Base/Table CSKH, token Lark.

🔑 Token & Base Info PAGE
GET /lark/token

Xem user access token, refresh token, app token, danh sách Bitable tables đã kết nối.

🦅 Lark Login Page PAGE
GET /lark

Trang đăng nhập riêng của Lark OAuth — dành cho cấu hình /lark/oauth/callback.

🚪 Đăng Xuất GET
GET /lark/logout

Xóa session, đăng xuất người dùng → redirect về /search-config.

🔄 Làm Mới Token Thủ Công GET
GET /lark/token/refresh

Kích hoạt TokenRefreshScheduler làm mới token ngay lập tức → redirect về /token.

🛡️ Admin Dashboard PAGE
GET /admin

Trang quản trị — tương tự trang chủ nhưng dành cho admin, kiểm tra token.

🔌 API Endpoints (JSON)
📡 Search Info 360° API GET
GET /api/search-info?phone=...

Tìm kiếm thông tin 360° khách hàng qua SĐT: Pancake POS + Lark Bitable.

📊 Sale Report Data API GET
GET /api/saleReport/data?range=

Trả JSON thống kê sale (CurrentMonth / LastMonth). Có cache session + disk.

📈 Stats Data API GET
GET /api/stats/data?customerMonth=

Dữ liệu thống kê lịch hẹn nhân viên: totalKhach, totalLich, totalHoanThanh.

🔄 Sync POS Orders API POST
POST /api/orders/sync

Đồng bộ đơn hàng từ Pancake POS vào DB. Body: startDate, endDate, status, pageSize.

👁️ Preview POS Orders API POST
POST /api/orders/preview

Xem trước danh sách đơn từ Pancake API mà không lưu vào DB.

🔍 Tra Cứu KH API API GET
GET /api/tra_cuu_khach_hang?phoneNumber=

API JSON tra cứu khách hàng song song trên nhiều Lark Base theo SĐT.

🚫 Sync Từ Chối Chăm API GET
GET /api/sync_tu_choi_cham

Đồng bộ khách hàng có "Tên Liệu Trình" = "Từ chối chăm" sang bảng đích.

💆 Sync Đang Chăm API GET
GET /api/sync_dang_cham

Đồng bộ khách hàng có "Tên Liệu Trình" = "Đang chăm" sang bảng đích.

💬 Lịch Sử Trao Đổi API GET
GET /api/exchanges?phone=

Lấy lịch sử trao đổi của khách từ Bitable table "Trao Đổi" theo SĐT.

🔧 Webhook Config Status API GET
GET /api/config/status

Kiểm tra trạng thái webhook status1 & status6 (bật/tắt).

📥 Export Sale Excel API GET
GET /saleReport/export?range=

Xuất file Excel (.xlsx) báo cáo sale tháng hiện tại hoặc tháng trước.

❤️ Health Check API GET
GET /api/orders/sync/health

Kiểm tra OrderSyncController và các service bean có hoạt động không.

⚡ Shortcut Sync TTC API GET
GET /updateTTC

Shortcut URL để chạy nhanh sync "Từ chối chăm" — alias của /api/sync_tu_choi_cham.

🔐 OAuth & Xác Thực
🔐 OAuth Callback GET
GET /oauth/callback

Lark OAuth callback sau khi người dùng authorize → lưu token → redirect về /.

🔒 Lark OAuth Callback GET
GET /lark/oauth/callback

Callback của Lark OAuth flow thứ 2 → tự động lấy App Token, Base IDs, Table IDs.

🔄 Config Refresh POST
POST /config/refresh

Làm mới toàn bộ dữ liệu config: All Bases, User Configs, Sale Tables.

♻️ Sale Report Refresh POST
POST /saleReport/refresh

Xóa cache sale report (session + disk) → tải lại dữ liệu mới từ Lark.
